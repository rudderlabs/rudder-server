Stale-dataset retries now refresh from in-memory DS snapshot, removing retry dependence on DB pool availability.
  • `UpdateJobStatusInTx` uses `dsList.snapshot()` under `RTryLockWithCtx`.
  • New compaction benchmark scenario adds deterministic seeding and 3 compaction strategy runs.
  • Migration/compaction lock-duration logs were added; stale-error noisy info logs were reduced.

Under high concurrency, stale DS retries previously needed a fresh DB-backed DS list read, but all pool connections could already be busy. This patch shifts retry refresh to the in-memory list maintained by compaction, so retries no longer require acquiring another DB connection.
