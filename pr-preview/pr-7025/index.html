TL;DR

Adds an ai-preview GitHub workflow and multi-stage Codex prompts to generate, publish, and tear down a visual PR review site.

  • Introduces .github/workflows/ai-preview.yaml with label-driven preview build/deploy and close-time teardown jobs.
  • Replaces single-prompt review-prompt.md flow with four stage prompts under .github/codex/agents/ and shared conventions.
  • Hardens workflow execution with app-token auth, explicit artifact checks, gh-pages bootstrap, and symlink-safe deployment copy.

Why

Reviewing unfamiliar PRs is slow; this PR automates a visual, single-page review briefing so reviewers can understand architecture, risks, and key hunks before reading raw diffs.

Risks

medium

Workflow gating is tightly coupled to action/label logic; future trigger expansion may silently skip intended rebuilds if not mirrored in preflight conditions.

Mitigation: Keep trigger list and preflight condition logic co-located and add explicit comments/tests in workflow docs for each supported action path.

medium

Stage contract drift between prompts and workflow checks could produce artifacts that pass one stage but fail assembly or deploy validation.

Mitigation: Version shared schema expectations in one source and add lightweight JSON shape validation immediately after each stage output.

low

Auto-initializing `gh-pages` with orphan commit may conflict with repositories that already manage branch protections or custom Pages history expectations.

Mitigation: Document branch bootstrap behavior and allow opt-out via workflow variable if organizations require pre-provisioned pages branch policy.

Review Checklist

  • Verify preflight outputs (`should_run`, `should_teardown`, `preview_path`) always align with each `pull_request_target` action path.
  • Confirm app-token permissions are minimum required for checkout, push to `gh-pages`, and Pages deployment.
  • Validate stage prompts only write declared files and never perform forbidden operations (tests/network/secrets output).
  • Ensure deploy step blocks unresolved asset references and does not copy symlinks into published preview content.
  • Check teardown job only runs on PR close and reliably removes the exact preview directory for the PR number.
