TL;DR

Adds a label-gated GitHub workflow and multi-stage Codex prompts to build, publish, comment, and clean up PR review preview sites on gh-pages.

  • Introduces .github/workflows/ai-preview.yaml that runs on pull_request labeled and closed events with label gate ai-preview.
  • Splits review generation into four Codex stages (analyze, diagrams, risks, assemble) coordinated via review-work/*.json artifacts.
  • Hardens workflow security by pinning actions and Codex CLI version, removing persisted checkout credentials, and redacting/normalizing metadata handling.

Why

Reviewing unfamiliar PRs is slow; this PR creates an opt-in, single-page visual review artifact so reviewers can quickly understand architecture, flow, risks, and key hunks before deep diff reading.

Pain: slow onboarding Fix: visual preview page

What changed at a glance

  • Codex review orchestrator prompt — Defines top-level multi-stage orchestration, artifact contracts, and non-interactive execution constraints for generating the review site. .github/codex/review-prompt.md

Risks

medium

Workflow still executes shell scripts over PR content and prompt files; malformed content could degrade output quality or expose logs if future edits weaken guards.

Mitigation: Keep strict quoting/redaction, pin dependencies, and periodically review shell safety around metadata extraction/comment rendering.

low

Preview deploy path cleanup assumes branch state consistency; manual gh-pages edits could leave stale preview artifacts.

Mitigation: Add periodic orphan sweep or scheduled integrity check for pr-preview/ directories.

low

Prompt-contract drift between stages can cause partial/empty sections without hard runtime failure.

Mitigation: Add schema validation for analysis.json/risks.json/diagrams.json before assemble step.

Review checklist

  • Confirm label gate and same-repo conditions prevent runs for forked PRs and unlabeled events.
  • Verify labeled path publishes preview and updates sticky comment idempotently on relabel.
  • Verify closed path removes both deployed directory and sticky comment without touching unrelated content.
  • Check that all action/Codex versions are pinned and checkout uses persist-credentials: false.
  • Validate prompt file rename is fully propagated and no references to removed review-promt.md remain.
  • Inspect metadata handling to ensure no sensitive strings from PR body/comments are emitted unredacted.
